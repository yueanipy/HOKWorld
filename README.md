# HOKWord — 《王者荣耀世界》黑盒视觉自动化

基于**屏幕画面识别 + 标准键鼠模拟**的自动化工具，当前实现**自动钓鱼**（抛竿 → 上钩 → 拉杆 → 收线 → 结算，含大鱼 QTE）与**实时剧情跳过**（识别剧情态自动跳过 / 推进，回到游戏即停手）。

> **纯黑盒**：只截屏 + 发标准键鼠输入；**不读写游戏内存、不注入、不改封包、不动任何游戏文件**。仅供学习研究，使用风险自负，请遵守游戏用户协议。

---

## 运行环境

- **Windows 10 / 11**（依赖 pywin32 / mss，仅支持 Windows）
- **Python 3.10+**
- 游戏《王者荣耀世界》以**窗口 / 无边框窗口**运行，且保持在**前台不被遮挡**（脚本靠截屏识别）
- **必须以管理员身份运行**：游戏通常提权运行，普通权限下 Windows UIPI 会拦截脚本的键鼠输入。脚本会在启动时自动请求 UAC 提权。

## 安装

```bat
git clone git@github.com:yueanipy/HOKWorldScript.git
cd HOKWorldScript
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

> 首次运行 `rapidocr-onnxruntime` 会自动下载 OCR 的 onnx 模型（约几十 MB），需联网。

## 运行

```bat
python app.py
```

- 启动会弹 **UAC 提权**（游戏需管理员发输入），点「是」。
- 在界面里点开「自动钓鱼」卡片的下拉箭头，可设「**循环次数**」「**完成后退出**」。
- 切到游戏、站在钓鱼点的**预备态**（已持竿、可抛竿），点「**开始**」，5 秒内切回游戏。
- **F12** 全局急停。

## 功能（自动钓鱼）

- 自动循环：抛竿 → 等待上钩 → 拉杆 → 收线 → 结算 → 续钓。
- **大鱼 QTE**：
  - *快速连点态*：跟随屏幕 A/D 方向**高频点按**消耗耐久（本作只能点按，按住会脱钩）。
  - *离散 QTE*：识别大圆按钮里的 **A/S/W/D** 字母，每个**随机延迟后按一次**。
- **记录鱼**：识别「个人新纪录 / F 放入背包」并处理。
- **落杆错误自动纠正**：识别「落点过近 / 超出落杆范围 / 落点不在水面 / 水域深度不足」——过近上移、过远下移、再不行重定位水域中央，连续失败则停机报原因。
- **等级上限**弹窗：按计划停机（不自动升级）。
- **抛竿看门狗**：连续多次空抛（疑似缺饵/朝向/钓点异常）→ 优雅停机告警。
- 结束输出统计：已钓数 / 抛竿数 / 用时 / 速率。

## 功能（剧情跳过 · 实时检测）

左侧「**实时检测**」页点「**开始**」后实时读屏，**每帧归类为一个状态再做唯一动作**（参考 better-genshin-impact / ok-wuthering-waves）：

- **可跳过剧情**（右上「跳过」控制条）→ 按 **ESC** 调出确认框 → 平滑移动光标点「跳过」按钮。
- **对话推进**（右上只有 抓拍/自动，或「点击空白处继续」）→ **原地点击**逐句推进。
- **黑屏过场** → 原地点击推进。
- **居中交互框**（如宝箱「点击开启」）/ **游戏世界**（右下技能 HUD 在）→ **完全不动作**。
- **「剧情结束即停手」保证**：靠「对话 UI 标志」+「右下 HUD 黏滞」门控，回到游戏世界立刻停止任何点击/按键，绝不在游戏里误点（=自动攻击）或误按 ESC 开菜单。
- 光标移动走**随机弧线 + 缓动**（非瞬移）；除点确认框按钮外全程不移动光标。

> 已知限制：**完全沉浸式**（控制条整段隐藏）的剧情暂不自动跳过。

## 目录结构

```
app.py                     Fluent 风格控制台（PySide6 + qfluentwidgets，自动提权）
recorder.py                数据录制器（采集连续帧+键鼠，供调参/补模板）
requirements.txt           依赖
fishing/
  matcher.py               识别（okww 式模板框架 + 二值字形 IoU + HoughCircles + OCR）
  fisher.py                钓鱼状态机（发真实键鼠）
  template_bank.py         命名模板 + 逐特征预处理（原图/二值/白掩膜）小框架
  replay_test.py           用录制会话回放验证识别
  templates/raw/*.png      识别模板（从真实客户端帧裁切，必需，勿删）
story/
  recognizer.py            剧情识别（OCR 控制条 + 右下 HUD 边缘密度 + 黑屏比例 → classify 状态）
  skipper.py               剧情跳过状态机（状态→动作；随机弧线移动光标，发真实键鼠）
```

## 注意事项

- 模板从 **1920 宽**的真实客户端帧裁切；脚本会把每帧归一化到 1920 宽再匹配。若你的分辨率 / 画质 / HUD 与采集环境差异很大，识别阈值可能需要按你机器重新标定（可用 `recorder.py` 录一段帧，再调 `fishing/matcher.py` 里的阈值/ROI）。
- 运行期会在 `sessions/` 下生成调试帧、在 `屏幕截图/` 下保存成功图，这些目录已在 `.gitignore` 中忽略。
- 仅在游戏前台时动作；切走会自动暂停。
