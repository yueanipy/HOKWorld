# HOKWorld — 《王者荣耀世界》黑盒视觉自动化

靠屏幕画面识别加标准键鼠模拟做三件事:**自动钓鱼**(抛竿、上钩、拉杆、收线、结算,带大鱼 QTE)、**剧情跳过**(认出剧情画面就自动跳过或推进,回到游戏立刻收手)、**跑图自动采集**(经过材料/宝箱出现「F + 名称」提示就识别并按 F 采集,按图标区分该不该采)。

只截屏、只发标准键鼠输入,不读写游戏内存、不注入、不改封包、不碰任何游戏文件。仅供学习研究,使用风险自负,请遵守游戏用户协议。

> **想直接用、不装 Python?** 到 [Releases](https://github.com/yueanipy/HOKWorld/releases) 下载免装安装版(当前 `v0.0.1`:`HOKWorldScript-0.0.1-Setup.exe`),双击安装即用。下面是从源码运行的说明。

## 运行环境

- Windows 10 / 11(x64),Python 3.10 及以上
- 游戏以窗口或无边框窗口运行,并且保持前台不被遮挡(全靠截屏认画面)

## 跑起来

```bat
git clone git@github.com:yueanipy/HOKWorld.git
cd HOKWorld
python -m venv .venv && .venv\Scripts\activate
pip install -r requirements.txt
python app.py
```

启动时会弹 UAC 提权(游戏是提权运行的,发键鼠得用管理员),点「是」。运行中 **F12** 全局急停。

## 配置与名单(都存程序目录 `data\`,更新 / 换机不丢)

脚本**一律真实操作游戏**(不再有「演练 / 真实输入」开关——它们会静默关掉操作、容易被误当失灵)。「时序抖动」默认关,可在「设置」开。

配置、日志、采集黑/白名单都写在**程序目录下的 `data\`**(`config.json`、`blacklist.txt`、`whitelist.txt`、`logs\hokworld.log` 等)——随程序、不进 Windows 用户目录(本程序不区分用户角色)。`data\` 不在安装器的文件清单里,所以 **覆盖升级 / 换台电脑更新都保留**你这台机器上的名单与开关;**卸载**时(控制面板或 uninstall)才把它连同程序一起清干净,且只动安装目录下这一处、绝不碰其它软件或盘符。

## 自动钓鱼

抛竿,等上钩,拉杆,收线,结算,接着续钓。大鱼会触发 QTE(快速连点 A/D,以及离散按钮 A/S/W/D);认出记录鱼就按 F 收进背包;落杆位置不对(过近、过远、不在水面、深度不够)会自动纠正;到等级上限或连续空抛太多次会优雅停机;结束打印一份统计。

## 剧情跳过(实时检测)

左侧「实时检测」点「开始」后实时读屏,每帧归到一个状态做唯一动作:

- 可跳过的剧情:ESC 调出确认框,再平滑移动光标点「跳过」
- 对话推进 / 点空白处继续 / 黑屏过场:原地点击
- 居中交互框(比如宝箱)、已经在游戏世界(右下 HUD 还在):不动

判断靠对话 UI 标志加右下 HUD 的黏滞门控,一回到游戏世界就立刻收手,不会误点、也不会乱按 ESC。沉浸式剧情默认不处理,需要的话可以在卡片里开「鼠标微动唤出控制条」(不太稳)。

## 跑图自动采集

「实时检测」里的「经过材料自动采集(F)」**默认开启**,和剧情跳过一起跑:跑图经过材料/宝箱时游戏弹「F + 名称」提示,脚本实时认出并按 F 采。

- **按图标分类**(快,主要不靠读字):F 键帽确认有提示 → 看键帽右侧图标 —— 手型(采集)/ 重现图标就采,NPC / 商店 / 制作 / 对话 / 组队等别的图标不动。
- **只在提示刚出现那一帧读一次名字**核对名单(同一提示不重复读,所以跟手);**重现**图标无歧义、免读字直接按。
- **碰撞名单(黑名单)**:图标也是采集手型、却不该采的交互(渡石、语印、捕获、启动滑索、松开滑索、启动冲云翼…)已**内置**,遇到自动跳过;误采了别的,到「设置 · 采集碰撞名单」加一行即可。
- **白名单(强制采)**:写进去的名字识别到就一定采(优先级最高,能盖过碰撞名单;也能强制采图标不是手型的东西)。
- 内置名单随版本更新自动生效;你自己增删的存程序 `data\`、更新不丢。

## 目录结构

```
app.py              Fluent 控制台(PySide6 + qfluentwidgets,含安全设置页,自动提权)
version.py          版本与应用信息
paths.py            资源 / 本机数据路径(数据在程序目录下的 data\)
config.py           配置(时序抖动;名单与配置都存程序目录 data\)
winenv.py           Windows 与游戏窗口工具(提权 / 隐藏控制台 / DPI / 窗口枚举 / 任务栏图标)
applog.py           统一日志(logs\hokworld.log)
requirements.txt    依赖
assets/             界面 / 应用图标
fishing/
  matcher.py        钓鱼识别(模板 + 二值字形 IoU + HoughCircles + OCR)
  fisher.py         钓鱼状态机(发真实键鼠)
  template_bank.py  命名模板 + 逐特征预处理
  templates/raw/*   识别模板(钓鱼 + 采集 F 键帽/手型/重现,从真实客户端帧裁切,必需,勿删)
story/
  recognizer.py     剧情识别(OCR 控制条 + 右下 HUD 边缘密度 + 黑屏比例)
  skipper.py        剧情跳过状态机(状态到动作,随机弧线移动光标)
gather/
  recognizer.py     采集识别(F 键帽 + 图标分类 + 文字兜底查名单)
  picker.py         采集引擎(边沿触发按 F,每个提示只读一次字)
  blacklist.txt / whitelist.txt   名单模板(首次运行复制到程序 data\,之后编辑 data\ 那份)
```

## 几点提醒

- 模板是从 1920 宽的真实客户端帧裁的,运行时每帧先归一化到 1920 宽再匹配。分辨率、画质、HUD 差太多时阈值可能要重标定,改 `fishing/matcher.py` 里的阈值和 ROI（也就是只适用于1080,1440,2160等16：9显示器）。
- 只在游戏前台时动作,切走会自动暂停。
